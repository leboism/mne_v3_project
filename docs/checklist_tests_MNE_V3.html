MNE Grade Manager V3

Checklist de tests — convergence vers une version utilisable

Priorité : Mac d'abord — Windows en dernier

Ordre recommandé :
  1. Un scénario bout-en-bout complet sur données de démo
  2. Le même scénario sur vos vrais fichiers Excel/PDF (2025-2026 ou 2026-2027)
  3. Les cas limites et fin d'année
  4. Windows en dernier (build + transfert zip)

Critère « prêt pour collègues » : les cases P0 sont toutes vertes, au moins 80 % des P1, et vous avez fait une sauvegarde/restauration sans perte.

Phase 1 — Socle (P0, à faire en premier)

Démarrage & millésime

  • Lancement : PYTHONPATH=src .venv/bin/python -m mne_grade_manager.main
  • Écran d'accueil : ouvrir le millésime 2026-2027 (ou celui que vous utilisez vraiment)
  • Ajouter un millésime manquant → il apparaît et s'ouvre
  • Retour à l'accueil puis réouverture → données intactes
  • Bandeau titre affiche le bon millésime

Sauvegarde / restauration (indispensable avant de confier l'outil)

  • Fichier → Exporter les données (transfert) → zip créé sans erreur
  • Modifier quelque chose (ex. un étudiant), Importer le zip → données restaurées, modification annulée
  • Vérifier qu'une photo ou PDF joint survit à l'export/import
  • Enregistrer la base SQLite seule → fichier cohérent

Données de démo (référence rapide)

  • Fichier → Créer des données de démo sur un millésime vierge
  • Parcourir tous les onglets sans crash
  • Repérer ce qui est « fictif » vs ce que vous attendez en prod

Phase 2 — Secrétariat (P0 / P1)

Étudiants

  • Ajouter un étudiant manuellement → visible dans la liste
  • Fiche : photo, pièces jointes, coordonnées → enregistrement OK
  • Modifier niveau / parcours / e-mail
  • Filtres : année, niveau, parcours, recherche nom, contrat péd., actifs/démissionnaires
  • Modèle d'import Excel → colonnes attendues
  • Importer Excel (votre fichier réel) → pas d'erreur bloquante, effectifs cohérents
  • Ré-import sur le même fichier → mise à jour sans doublons aberrants
  • Exporter Excel de la liste filtrée → ouvre dans Excel
  • Démissionnaire puis filtre « Démissionnaires » → Réintégrer
  • Liste d'e-mails → adresses exploitables pour un mail groupé
  • Importer dossiers candidature (PDF) sur 2–3 vrais PDF → champs extraits plausibles

Maquette

  • Importer la maquette M1 P, M1 C, M2 (fichiers officiels)
  • Vérifier UE, ECTS, heures après import
  • Dupliquer ou reporter vers l'année suivante (ne pas modifier l'ancienne)
  • Exporter Excel → fichier lisible
  • Après import maquette : onglet Cours reflète les UE

Emploi du temps

  • Import M1 (Physique + Chimie) → grille, couleurs, heures en bas
  • Navigation S1 ↔ S2 (M1)
  • Import M2 → parcours Common (X) forcé
  • M2 : S1, S2 (janv.–juin), S3 (été) — même si S3 vide pour l'instant
  • Fenêtre de semaines ◀ ▶ si beaucoup de colonnes
  • Ré-import remplace l'ancien (même millésime/niveau)
  • Supprimer l'emploi du temps puis ré-importer
  • Écart heures planifiées / prévues : valeurs plausibles ou « — » si pas de référentiel

Équipe du master

  • Fichier → Équipe du master : directeurs, responsables parcours, secrétariats
  • Enregistrement → rouvrir la fenêtre → données conservées

Phase 3 — Coordination pédagogique (P1)

Cours

  • Arborescence M1/M2 → fiche UE (enseignant, syllabus)
  • Lien maquette ↔ cours cohérent après imports

Notes

  • Sélection maquette / étudiant / cours / évaluation
  • Saisie de notes → moyenne recalculée
  • Navigation clavier / passage ligne suivante
  • Message ou garde-fou si non enregistré avant changement de sélection
  • Cas ABI / ABJ / DEF / session 2 si vous les utilisez cette année

Résultats

  • Tableau synthèse par étudiant après saisie de notes
  • Filtres niveau / parcours / validation
  • Export CSV → exploitable (Excel, séparateur, encodage accents)

PV & délibérations

  • Créer une délibération S1, S2, Finale pour un parcours test
  • Composition du jury (manuel + import Excel modèle si vous l'utilisez)
  • Délibération : décisions par étudiant
  • M1 : Admis M2 / Redoublement → Appliquer sur la fiche
  • M2 : Année validée → statut diplômé sur la fiche
  • Action groupée Appliquer progressions M2 / redoublement / diplômé
  • Exports PDF : PV, matrice de notes, relevé — fichiers non vides, mise en page acceptable
  • Passage M2 / redoublement (menu Étudiants) cohérent avec le jury

Statistiques

  • Effectifs, nationalités, taux de réussite : chiffres plausibles
  • Export CSV

Convocations d'examen

  • Sélection cours / étudiants
  • Objet mail avec code MNE
  • Ouvrir dans l'app mail → destinataires en Cci (Mail sur Mac)

Phase 4 — Scénarios bout-en-bout (P0 avant juillet)

Début d'année

  1. Ouvrir millésime
  2. Exporter (archive de départ)
  3. Importer étudiants
  4. Importer maquettes M1 + M2
  5. Importer emplois du temps
  6. Compléter équipe du master
  7. Vérifier effectifs et 5 fiches étudiants au hasard

Milieu d'année

  1. Mise à jour Excel étudiants (changement e-mail / parcours)
  2. Ré-import emploi du temps révisé
  3. Saisie notes session 1 sur 2 UE
  4. Export CSV résultats + contrôle manuel des moyennes

Fin d'année

  1. Délibération S1 puis S2 ou finale
  2. Appliquer progressions
  3. Marquer démissionnaires définitifs
  4. Export transfert final
  5. Rouvrir l'app → tout est là

Phase 5 — Robustesse (P2, quand le métier est OK)

  • Fermer l'app au milieu d'une saisie de notes → pas de corruption à la réouverture
  • Gros import (tous les étudiants + toutes les maquettes) : temps acceptable, pas de freeze prolongé
  • Caractères spéciaux : noms avec accents, apostrophes, e-mails
  • Millésime sans maquette M2 → message d'erreur clair au passage M2
  • Onglet vide (pas d'import EdT) → message utile, pas d'écran blanc
  • Tout actualiser après gros import → listes à jour partout

Phase 6 — Windows (plus tard, liste courte)

À ne faire qu'après que les P0 Mac soient verts :

  • Build scripts/build_windows.ps1 sur une machine Windows
  • Double-clic MNEGradeManager.exe → accueil
  • Export zip sur Mac → Import sur Windows (ou l'inverse)
  • Chemins C:\Users\...\.mne_grade_manager\
  • Outlook / client mail par défaut pour convocations
  • Affichage écran 1366×768 ou petit laptop
